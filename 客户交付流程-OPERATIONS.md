# enkiseojp.com 客户交付流程（运营 SOP）

> **现状一句话**：现在的 app 是「**单人运营的规划台 + 真实 AI 产文 + 自营博客**」，
> **不是**带支付和账号的自助 SaaS。所以"客户付费后的流程"今天 =
> 一套**你来操作的代运营流程（done-for-you）**，不是系统自动跑。
> 下面分「现在就能跑的手动流程」和「要做成自动化还缺什么」。

---

## 0. 先认清产品现在到哪一步（重要，别当成已经全自动）

| 模块 | 现状 | 说明 |
|---|---|---|
| 支付 Stripe | ❌ 没接 | app 里只是文案"Stripe/銀行振込"，**收款全靠手动** |
| 客户账号/登录 | ❌ 无 | 状态只存浏览器 `localStorage`，换台电脑就没了 |
| 客户后台（多租户） | ❌ 无 | 现在的 3 个页面是**你自己**管关键词用的，不是给客户登录的 |
| AI 产文 | ✅ 真实 | DeepSeek V4 Pro（`/api/generate-article` + `scripts/gen-article.mjs`），输出草稿，**手动复制** |
| GSC 排名追踪 | ⚠️ 单站 | `GSC_SITE_URL` 只能配一个站（现在=enkiseojp.com），查客户站要换/手动 |
| 自营博客 | ✅ 自动 | 每周二 DeepSeek 自动发到 enkiseojp.com/blog |

**结论**：前几个付费客户，按**手动代运营**扛，别等系统。下面第 1、2A、3 节就是手动 SOP；第 2B 是以后要不要做成 SaaS 的清单。

## 1. 客户付费后的整体走流程（end-to-end）

标注：【你做/手动】【app/自动】【客户做】

1. **询盘 → 报价**【你】：按目标关键词难易给档
   - 🟢 かんたん：3 个月冲首页 / **¥3,300 税込/月**
   - 🟡 ふつう：6 个月 / **¥6,600 税込/月**
   - 🔴 むずかしい：10 个月 / **¥13,200 税込/月**
2. **收款**【手动】：Stripe Payment Link 或 银行转账。月付或一次付清都行（SEO 节奏不变）。开发票（特商法/インボイス）。
   - ＊你说的"等一个月付清全款"：产品是按**月**计费模型；要做一次性付清整期也行，交付节奏一样按月跑。
3. **启动会 / intake**【你+客户】：必须拿到这 **5 样**，缺一不可——
   1. 客户**网站 URL**
   2. **目标关键词**（先 1〜3 个）
   3. 客户 **GSC 属性的访问**：把你的服务账号邮箱（SA email）加为该属性的"用户"。**否则查不了他的排名**
   4. 客户**网站发布权限**：WordPress/CMS 账号，或约定"我们给稿、你来发"
   5. **E-E-A-T 素材**：公司信息、负责人姓名、营业执照/特商法信息、**一条真实案例或数据**
4. **建项目**【你/app】：在 app 里 add 关键词（自动判 KD → 定档 → 生成月度任务表），或用表格管理。
5. **月度执行循环**【你/app】：按档跑月度任务（见第 3 节）。
6. **月次报告**【你】：GSC 排名变化 + 本月发了几篇 + 下月计划 → 发客户。
7. **续费 / 解约**：月付模型，解约从次月起。

## 2. 后台需要配置的内容

### A. 现在就要配的（Vercel + Google，一次性）

**Vercel 项目 `jp-seo-bot` → Settings → Environment Variables：**

| 变量 | 值 | 状态 |
|---|---|---|
| `DEEPSEEK_API_KEY` | DeepSeek 的 key | ✅ 已配 |
| `GSC_SA_KEY_B64` | `base64(服务账号 JSON key 全文)` | 要配 |
| `GSC_SITE_URL` | 要追踪的 GSC 属性，如 `sc-domain:客户域名` 或 `https://客户域名/` | 要配 |

**Google Cloud / GSC 侧：**
1. 建一个服务账号（SA），开启 **Search Console API**。
2. 下载 SA 的 JSON key → `base64` → 填进 `GSC_SA_KEY_B64`。
3. 把 SA 邮箱加到**客户的** GSC 属性 → 用户和权限 → 加用户（读取即可）。
4. 给客户站提交 sitemap、验证属性。

> ⚠️ **关键限制**：`GSC_SITE_URL` 只有**一个值** → 一个部署只能自动追踪**一个站**。
> 多个付费客户时：要么每个客户**手动**在 GSC 网页看排名（不走 app），
> 要么改成多租户（见 2B）。前几个客户建议手动。

### B. 要做成「真·自助 SaaS」还缺的（要决定建不建）

**只有当客户 > ~5 个、手动扛不住时才值得建。现在 0 付费客户，先别建。**

- 客户登录/账号（Supabase Auth）
- 多租户数据库（`customers / projects / keywords / articles / ranks` 表）—— 取代现在的 localStorage（换设备就丢）
- Stripe 订阅 + webhook → 付款成功自动开通、退款/逾期自动停用
- 每客户**独立 GSC**：用 OAuth 让客户自己授权，而不是单个 env
- 文章**自动发布**到客户站（WordPress REST API 等），取代手动复制

> 这是**几周的开发量**，不是配置。先用手动代运营验证商业模式，跑通了再投。

## 3. SEO 的具体操作步骤（每月循环）

不同档篇数不同，但步骤一样：

1. **关键词调研**：定本月主关键词 + 2〜3 个长尾。app 自动判 KD 定档。
2. **产文**【DeepSeek V4 Pro】：
   - 自营博客 → `node --env-file=supabase/.env scripts/gen-article.mjs "关键词" "切口" "slug" --write`
   - 客户站草稿 → app 里点「今月の AI 記事を生成」，或同脚本
   - 篇数：🟢 2 / 🟡 4 / 🔴 8 篇/月
3. **编辑门**【必做，不能省】：每篇把正文里的 `[体験談/データを追記]` 换成**一条真实一手数据/案例**。
   这是防 Google「AI 内容农场」惩罚的关键 —— 纯 AI 文第 3 个月会崩，加一手真实信息才稳。
4. **发布**【手动】：贴到客户网站（WordPress 等），带上 app 给的 **JSON-LD 结构化数据**（Article + FAQ），设 canonical。
5. **站内优化**：内部链接（新文链到相关老文）、标题/meta description 优化。
6. **站外**【🟡🔴】：medium 起做被链接获取、はてな/note 投稿；🔴 hard 每月 1 篇 **PR TIMES** 配信。
7. **监控 + 报告**【你/app】：GSC 看排名（app 的「GSC で順位取得」按钮，需先配 2A 的 env），月底出报告。

### 各档月度任务模板（app 内置，对照用）

- 🟢 **easy（3 个月）**：每月 AI 记事 2 本 + 内部链接整理 / 关键词簇扩张 / 排名验证。
- 🟡 **medium（6 个月）**：每月 4 本 + 被链接候选 / はてな・note 投稿 / 比较站申请 / 既存改写。
- 🔴 **hard（10 个月）**：每月 8 本 + PR TIMES（月 1）+ Qiita/Zenn + AI Overview 对应（FAQ/Schema）+ 既存改写。
